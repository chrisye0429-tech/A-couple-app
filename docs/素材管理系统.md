# 素材管理系统

## 当前目标

素材管理系统负责统一管理手账编辑器里的贴纸、字体和背景。当前阶段已实现核心数据模型、JSON 索引、分页查询、分类缓存、图片预取，以及字体文件下载缓存和 Expo 字体注册能力，暂不接入真实素材商城或后台管理系统。

## 代码位置

- `src/features/materials/types.ts`：素材数据模型定义。
- `src/features/materials/assetCdnManager.ts`：素材 CDN 版本检查、增量下载和本地缓存状态管理。
- `src/features/materials/materialIndex.json`：素材 JSON 索引配置。
- `src/features/materials/stickers.json`：贴纸库 JSON 配置，当前包含 20 个分类、100 个示例贴纸。
- `src/features/materials/fonts.json`：字体库 JSON 配置，当前包含 20 款免费可商用字体。
- `src/features/materials/materialManager.ts`：素材管理器核心逻辑，负责分页、分类缓存、图片预取、字体下载缓存和字体注册。
- `src/features/materials/index.ts`：素材模块统一导出入口。

## 素材类型

### 贴纸 Sticker

贴纸素材包含：

- `imageUrl`：贴纸原图地址。
- `category`：分类，例如 love、pet、weather。
- `tags`：标签，例如爱心、情侣、萌宠。
- `thumbnailUrl`：缩略图地址。
- `provider`：素材来源，当前支持 `iconify`、`flaticon`、`emoji`。
- `status`：素材状态，`ready` 表示可用，`placeholder` 表示当前为占位素材，后续需要替换。

当前贴纸库包含以下 20 个分类：

- 植物类。
- 动物类。
- 装饰类。
- 情绪类。
- 季节类。
- 美食类。
- 旅行类。
- 文具类。
- 边框类。
- 爱情类。
- 纪念日类。
- 天气类。
- 星座类。
- 音乐类。
- 运动类。
- 家居类。
- 节日类。
- 学习类。
- 工作类。

贴纸 URL 优先使用 Iconify 公共 API。Flaticon 不直接热链进 App；如后续要使用 Flaticon，需要先确认下载、商用、署名和再分发规则。

### 字体 Font

字体素材包含：

- `fontFileUrl`：字体文件地址。
- `fontName`：注册到 App 内使用的字体名称。
- `supportedLanguages`：支持语言，例如 `zh-Hans`、`en`。
- `previewText`：字体预览文案。
- `fallbackFontFamily`：未完成下载或注册时的跨平台备用字体。
- `recommendedLineHeight`：建议行高，降低不同平台渲染差异。
- `status`：`ready` 表示路径可优先接入，`pending-review` 表示上线前需要复核 CDN 直链和授权页面。

当前项目已接入 `expo-font`。字体被用户选择预览时，会先通过 `expo-file-system` 下载到本机缓存目录，再使用 `ExpoFont.loadAsync` 注册到 App 内。注册成功后，文字元素可直接使用字体配置里的 `fontName` 作为 `fontFamily`。

### 背景 Background

背景素材包含：

- `imageUrl`：背景图地址。
- `backgroundType`：背景类型，支持 `solid`、`gradient`、`texture`、`pattern`。
- `previewUrl`：预览图地址。
- `color`：主背景色，纯色背景直接使用该颜色绘制。
- `secondaryColor`：纹理或图案辅助色。
- `accentColor`：纹理或图案强调色。
- `renderMode`：本地绘制模式，例如 `grid`、`dot`、`lined`、`kraft`、`watercolor`、`floral`、`star`。
- `fitMode`：背景适配方式，当前支持 `tile` 和 `cover`。
- `customizable`：是否允许作为自定义纯色的基础模板。

当前背景库包含 20 个本地可绘制模板：

- 纯色系 8 种：奶茶色、雾霾蓝、豆沙粉、抹茶绿、姜黄色、高级灰、薰衣草紫、珊瑚橙。
- 纹理类 6 种：网格纸、点阵纸、横线纸、牛皮纸纹理、水彩晕染、复古做旧。
- 图案类 6 种：碎花、波点、条纹、格子、云朵、星星。

当前背景不依赖外部图片下载。`imageUrl` 使用 `local-background://` 占位协议，实际由编辑器根据 `renderMode` 和颜色字段在本地绘制。

## 加载策略

- 首屏加载 20 个常用素材：按 `priority` 从高到低取前 20 个。
- 分页加载更多：默认每页 20 个，可通过 `page` 和 `pageSize` 调整。
- 分类缓存：按 `kind + category` 保存已缓存素材 id 列表，方便后续快速打开同一分类。
- 图片缓存：贴纸和远程背景使用 React Native `Image.prefetch` 预取网络图片；本地绘制背景不会发起网络请求。
- 字体文件缓存：字体使用 `expo-file-system` 的 `Paths.cache`、`Directory` 和 `File.downloadFileAsync` 下载到本机缓存目录 `couple-journal-fonts`。
- 字体注册：字体下载完成后，素材管理器默认调用 `expo-font` 的 `loadAsync` 注册字体；同一字体已注册时会跳过重复注册。
- 旧缓存兼容：如果历史缓存里只保存了字体网络 URL，素材管理器会视为未完成文件缓存，并重新下载到本地。

## 使用示例

```ts
import { materialManager } from './features/materials';

const firstScreenMaterials = materialManager.getFirstScreenMaterials();
const stickerPage = materialManager.listMaterials({
  kind: 'sticker',
  category: 'love',
  page: 1,
  pageSize: 20,
});
const stickerCategories = materialManager.listStickerCategories();
const fontPage = materialManager.listFonts({
  page: 1,
  pageSize: 20,
});
const preview = materialManager.getFontPreviewConfig('font-lxgw-wenkai');

await materialManager.cacheFirstScreenMaterials();
await materialManager.cacheCategory('sticker', 'love');
await materialManager.prepareFontForPreview('font-lxgw-wenkai');
```

默认素材管理器已经接入 Expo 字体注册：

```ts
import { materialManager } from './features/materials';

await materialManager.prepareFontForPreview('font-lxgw-wenkai');
```

如后续测试或特殊环境需要替换注册方式，也可以自定义 `FontRegistrar`：

```ts
import * as Font from 'expo-font';
import { MaterialManager, createExpoFontRegistrar } from './features/materials';

const manager = new MaterialManager({
  fontRegistrar: createExpoFontRegistrar(Font.loadAsync),
});
```

字体预览推荐流程：

1. 先用 `listFonts` 展示字体列表。
2. 用 `getFontPreviewConfig` 生成预览文案、字体名、备用字体和行高。
3. 用户点选某个字体时，调用 `prepareFontForPreview` 下载到本地缓存并注册字体。
4. 注册完成后，把文字元素的 `fontFamily` 设置为对应 `fontName`。
5. 如果下载失败，保持 `fallbackFontFamily`，并提示稍后重试。

## 后续接入方向

- 编辑器贴纸面板改为从 `MaterialManager` 读取贴纸素材。
- 背景面板改为从 `MaterialManager` 读取背景素材。
- 文字样式面板增加字体选择，并调用字体注册能力。
- 文字元素数据结构后续增加 `fontFamily` 字段，保存用户选择的字体。
- 后续需要在 iOS 和 Android 模拟器上逐项验证中文字体格式兼容性，优先使用 `ttf` 或 `otf` 文件。
- 素材 CDN 发布与增量更新按 `docs/素材CDN部署方案.md` 执行，App 端使用 `checkAndDownloadAssetUpdates` 检查远程 `version.json`。
