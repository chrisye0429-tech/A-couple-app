# 设计规范

## 总体风格

UI 应清新、简洁、温柔，但不要过度花哨。界面重点是帮助用户轻松制作手账，而不是用复杂装饰抢走内容注意力。

## 视觉原则

- 背景干净，留白充足。
- 控件清楚易懂，适合新手使用。
- 颜色柔和，但不能整套界面只有单一色系。
- 圆角适中，不使用过度夸张的卡片堆叠。
- 文本必须清晰可读，不能和背景或贴纸混在一起。

## 主题色规范

首版可规划以下色系：

- 樱粉色系：温柔、轻松。
- 薄荷绿色系：清新、干净。
- 奶蓝色系：安静、柔和。
- 暖黄色系：明亮、治愈。

主题色只影响 App 的导航、按钮、工具栏、标签等外层界面。用户已经制作好的手账页面内容不应因为切换主题而改变。

## 手账编辑器规范

编辑器是核心体验，首版需要做到：

- 画布区域清楚，用户知道哪里是手账页面。
- 工具栏简单，包含背景、文字、贴纸、分页、收藏、图片等基础入口。
- 底部点选按钮使用“图标 + 标题”形式，不使用长说明文字占据按钮空间。
- 选中元素后可以移动、缩放、删除。
- 选中元素后，优先在元素四角提供快捷操作点：左上角删除、右上角旋转、左下角透明度、右下角复制。
- 四角操作点应使用圆形按钮，按钮需要清晰但不能遮挡太多内容。
- 画布上方应提供“回退 / 前进”按钮，帮助用户找回误删、误改或误移动前的页面状态。
- 回退和前进按钮不可用时应有禁用状态，避免用户误以为点击无效是故障。
- 双指缩放应以元素中心为基准，避免缩放时元素突然跳动。
- 默认拖动元素时不自动吸附，元素应跟随手指精确移动；“对齐辅助”作为独立开关，开启后才出现轻量辅助线并吸附对齐。
- 文字编辑入口要明显，不让用户猜；当前使用“文字内容”面板进行多行编辑。
- 文字元素在画布上只显示文字本身，不默认显示边框和底色，避免破坏手账排版。
- 文字样式应至少支持字体、字号、颜色、加粗和斜体，入口放在文字内容面板内。
- 贴纸面板应直接读取素材库分类，不应只保留少量硬编码示例贴纸。
- 保存反馈要清楚，例如显示“已保存”。

## 信纸规范

标准信纸首版提供 20 种基础模板：

- 手机端编辑信纸采用竖版 4:3 比例，高宽对应为 400 x 300；基础画布尺寸为 300 x 400，方便在手机屏幕内查看完整页面。
- 纯色系 8 种：奶茶色、雾霾蓝、豆沙粉、抹茶绿、姜黄色、高级灰、薰衣草紫、珊瑚橙。
- 纹理类 6 种：网格纸、点阵纸、横线纸、牛皮纸纹理、水彩晕染、复古做旧。
- 图案类 6 种：碎花、波点、条纹、格子、云朵、星星。
- 纯色背景应直接用颜色对象绘制，不依赖图片。
- 纹理和图案优先用本地组件绘制，后续如换成图片素材，应支持平铺或缩放适配。
- 背景面板应支持自定义纯色，当前以常用色板和十六进制颜色输入作为取色器原型。

信纸应作为页面背景，不遮挡用户添加的图片、文字和贴纸。

## 贴纸规范

贴纸首版以轻量素材为主：

- 爱心。
- 五角星。
- 萌宠。
- 太阳。

贴纸添加到画布后应按透明 PNG 风格显示，不带默认底色、圆形底或按钮底。贴纸可以拖拽、缩放和删除。首版不做复杂贴纸商城。
