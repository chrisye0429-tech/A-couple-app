# 验收标准

## 通用验收标准

每次开发完成后，都要检查：

- 功能是否能从开始到结束完整使用。
- 是否有明显报错、卡死或打不开页面。
- 是否影响了已有功能。
- 文档是否需要同步更新。
- 当天开发日志是否已更新。

## 阶段 1：项目基础

完成标准：

- 存在 `AGENTS.md`。
- 存在 `docs/需求说明.md`。
- 存在 `docs/技术方案.md`。
- 存在 `docs/设计规范.md`。
- 存在 `docs/开发步骤.md`。
- 存在 `docs/验收标准.md`。
- 存在当天开发日志。
- 文件内容均为中文，并能指导后续开发。

## 阶段 2：本地手账编辑器

完成标准：

- 用户可以创建一页手账。
- 用户可以选择信纸背景。
- 用户可以添加图片、文字和贴纸。
- 手账信纸在手机端按竖版 4:3 比例显示，高宽对应为 400 x 300，当前基础尺寸为 300 x 400。
- 底部工具入口应显示为图标加标题，能够快速识别背景、文字、贴纸、分页、收藏和图片。
- 页面元素可以拖拽移动。
- 页面元素可以缩放，缩放后尺寸显示在属性面板中。
- 页面元素可以旋转，旋转后角度显示在属性面板中。
- 画布上方应有“回退 / 前进”按钮；没有可撤销或可恢复内容时，按钮应显示禁用状态。
- 用户误删元素后，点击“回退”应恢复被删除的元素；再点击“前进”应重新回到删除后的状态。
- 拖拽移动和双指缩放应按一次手势记录一步撤销，不能每移动一小段都变成一步撤销。
- 选中元素后，四角应出现快捷操作点：左上角删除、右上角旋转、左下角透明度、右下角复制。
- 点击透明度操作点后，元素透明度应变化，并在属性面板显示当前百分比。
- 点击复制操作点后，应新增一个相同元素并自动选中新元素。
- 用户可以修改文字元素内容，修改后画布和属性面板应同步显示。
- 用户可以修改文字字号、颜色、加粗和斜体，修改后画布应同步显示。
- 贴纸素材至少包含爱心、五角星、萌宠、太阳，并能添加到画布继续编辑。
- 页面可以保存并再次打开继续编辑。
- 拖拽后的元素位置应更新到属性面板，并能通过本地保存保留下来。
- 缩放后的元素尺寸应能通过本地保存保留下来。
- 旋转后的元素角度应能通过本地保存保留下来。
- 默认拖动元素时应跟随手指自由移动，不自动吸附；打开“对齐辅助”后，靠近其他元素或画布中心时才显示辅助线并吸附。

## 阶段 3：文件夹与页面管理

完成标准：

- 用户可以创建和命名文件夹。
- 用户可以重命名和删除文件夹。
- 用户可以在文件夹中创建手账页面。
- 用户可以打开、编辑和保存已有页面。
- 文件夹和页面列表显示清楚。

## 阶段 4：账号与情侣绑定

完成标准：

- 用户可以注册或登录。
- 用户可以生成邀请码。
- 另一方可以输入邀请码完成绑定。
- 绑定后双方进入同一个情侣空间。

## 阶段 5：同步共同编辑

完成标准：

- 情侣双方可以看到同一批文件夹和页面。
- 一方保存修改后，另一方可以看到更新。
- 非绑定用户不能编辑情侣空间内容。
- 基本处理同时编辑时的数据覆盖风险。

## 阶段 6：只读分享

完成标准：

- 可以生成只读分享链接。
- 第三方打开链接后可以查看内容。
- 第三方不能编辑、删除或复制编辑权限。
- 分享展示效果和 App 内页面一致。
