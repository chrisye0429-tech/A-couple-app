# AGENTS.md

本文件是项目协作与开发工作说明。后续每次在本项目中工作时，都必须先阅读本文件，再按 `docs/` 中的标准文档推进。

## 沟通语言

- 始终使用中文与用户沟通。
- 项目文档、开发日志、提交说明、错误解释和建议均使用中文。
- 代码注释和说明文档默认使用中文，除非第三方框架或接口要求英文。

## 项目定位

本项目是一款情侣互动手账 App，目标是一套代码同时支持 iOS 和 Android。用户可以使用信纸、图片、文字和贴纸制作手账页面，并通过情侣邀请码共享、共同编辑，也可以生成只读链接分享给第三方查看。

## 标准文件路径

- `docs/需求说明.md`：记录产品目标、用户角色、首版范围、暂不做事项。
- `docs/技术方案.md`：记录跨平台技术路线、前后端边界、数据保存方向和技术原则。
- `docs/设计规范.md`：记录 UI 风格、主题色、编辑器、信纸、贴纸和交互规范。
- `docs/开发步骤.md`：记录分阶段开发顺序，避免一次做太多。
- `docs/验收标准.md`：记录每个阶段如何判断功能完成、稳定、可继续推进。
- `docs/运行说明.md`：记录项目安装、检查、启动方式和本机运行注意事项。
- `docs/首页与导航说明.md`：记录当前首页、底部导航和基础页面结构。
- `docs/手账编辑器数据结构.md`：记录手账页面、元素、工具栏和编辑器页面结构。
- `docs/素材CDN部署方案.md`：记录素材 CDN 目录、发布脚本、GitHub Pages、jsDelivr 和 App 端版本检查方案。
- `development-logs/`：记录每日开发完成事项、问题、待办和风险。

## 工作流程

1. 开发前先阅读 `AGENTS.md` 和 `docs/` 下相关文件。
2. 每次只推进一个小阶段，优先保证项目稳定、清晰、可回退。
3. 实现功能前先确认当前阶段目标，不随意扩大范围。
4. 完成开发后更新当天开发日志；如果当天日志不存在，则创建当天日期文件。
5. 如果开发中发现需求、技术或设计标准需要调整，必须同步更新 `docs/` 中对应文件。
6. 不在没有说明的情况下删除用户已有内容或重置项目文件。

## 每日开发日志规则

日志文件命名格式为 `development-logs/YYYY-MM-DD.md`。

每日日志必须包含：

- 今日完成
- 遇到问题
- 明日待办
- 风险提醒

当前阶段的“每天自动记录”定义为：每次开发任务结束时自动维护当天日志。未来如果需要固定时间提醒或定时生成日志，再增加自动化任务。

## 当前首版默认方向

- App 类型：情侣共同手账 App。
- 首版重点：共同手账体验。
- 登录方式：手机号或邮箱。
- 协作方式：情侣双方同步编辑。
- 技术方向：React Native / Expo + TypeScript。
- 设计方向：清新、简洁、可切换色系，不影响内部内容。
