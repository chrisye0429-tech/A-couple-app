# 2026-05-25 开发日志

## 今日完成

- 修复问题 #1：元素拖动行为不符合预期。
- 将编辑器拖拽逻辑改为默认自由移动，元素会跟随手指位置更新，不再默认吸附到附近元素或画布中心。
- 新增“对齐辅助”开关，默认关闭；只有用户主动开启后，才会调用吸附计算并显示辅助线。
- 关闭“对齐辅助”时会清除当前辅助线，避免残留视觉提示。
- 更新 `docs/验收标准.md`、`docs/设计规范.md` 和 `docs/手账编辑器数据结构.md`，将默认行为改为自由拖动。
- 继续修复问题 #1：选中元素后的四角操作热区过大，小贴纸几乎整块都会被误判为删除、旋转、透明度或复制，已改为仅在圆形按钮附近响应操作，中间区域保留给拖拽。
- 修复问题 #2：元素选中后的四角操作菜单顺序错误。
- 将四角操作点调整为：左上删除、右上旋转、左下透明度、右下复制。
- 将四角操作点显示文案改为对应图标：删除、旋转、透明度、复制，避免只看到透明度相关控件。
- 同步更新 `docs/验收标准.md`、`docs/设计规范.md` 和 `docs/手账编辑器数据结构.md` 中的四角操作顺序。
- 定位问题 #3：贴纸点击后其实已进入页面数据，但素材库大量使用远程 SVG 地址，当前 React Native 图片渲染容易显示为空白，导致看起来像没有添加成功。
- 修复问题 #3：远程 SVG 贴纸暂时改用透明 emoji/文字兜底显示，避免新增后不可见。
- 优化贴纸新增位置：点击贴纸后会在信纸中心附近出现，初始大小约为信纸宽度的 20%，并立即选中，方便继续拖拽和调整。
- 定位问题 #4：字体选择后文字外观无变化，主要风险来自远程字体未注册时不更新文字状态，以及自定义字体渲染时仍叠加强制粗体，容易回退为系统字体。
- 修复问题 #4：切换字体会写入选中文字元素并触发页面状态更新；远程字体注册失败时会使用本机替代字体兜底。
- 修复问题 #4：画布渲染自定义字体时不再强制叠加粗体和斜体，避免单文件字体被系统回退为默认字体。
- 修复问题 #5：编辑器不再放在 App 外层整页滚动容器中，避免底部素材面板滚动时带动整个编辑页。
- 修复问题 #5：编辑器内部新增底部独立滚动区，工具栏、背景、贴纸、文字、保存、图层和属性面板在下方滚动，上方手账预览区域保持固定。
- 继续修复问题 #5：上方手账预览区也改为独立滚动区，避免预览区过高占满屏幕后无法触达下方工具面板；现在预览区和工具区各自滚动、互不干扰。
- 修复问题 #6：底部面板改为按当前工具过滤内容，背景、文字、贴纸、图片、收藏和分页各自显示对应面板，不再混合渲染。
- 修复问题 #6：图片工具不再点击 tab 后立即打开相册，而是在图片面板中提供“选择图片”入口。
- 重构底部工具栏第一步：从原 `ToolBar` 抽出 `EditorBottomToolBar` 和 `EditorToolIcon`，保持现有点击逻辑、样式和显示效果不变，为后续 Bottom Sheet 交互做准备。
- 补齐底部工具栏父级状态同步：新增工具面板开关状态，点击相同工具时收起面板，点击不同工具时打开并切换内容。
- 实现真正的底部弹出面板结构：底部工具栏常驻，素材面板通过 `EditorBottomSheet` 从底部动画弹出，支持遮罩点击关闭和向下拖拽关闭。
- 调整底部层级：预览区占据主要空间，保存状态保留在主界面，素材面板浮层显示在工具栏上方，不再作为固定内容挤占主页面。
- 迁移素材面板到 `EditorToolPanel`：`EditorBottomSheet` 只负责弹出层、动画和滚动，背景、贴纸、文字、图片、收藏和分页内容统一由 `EditorToolPanel` 根据当前工具渲染。
- 完善 Bottom Sheet 拖拽关闭细节：拖拽开始会停止旧动画，向下拖动会限制最大位移，未达到关闭阈值会回弹，被系统打断时也会回弹。
- 完善底部工具栏状态同步：面板关闭时工具按钮不显示选中态，面板打开或切换工具时再同步高亮当前工具。
- TypeScript 类型检查通过。

## 遇到问题

- 旧文档中仍保留“默认自动吸附”的描述，已同步修正。
- 模拟器复测发现自由拖动仍受影响，进一步定位为四角操作点命中范围过大，而不是对齐吸附仍在开启。
- 旧文档中仍保留旧版四角操作顺序，已同步修正。
- 贴纸库当前使用的 Iconify 地址多为 SVG，暂未接入 SVG 专用渲染库，因此先用 emoji/文字兜底保证编辑流程可用。
- 部分远程字体可能因为网络、CDN 路径或字体文件格式暂时无法注册，当前先提供本机替代字体兜底，后续需要继续核对字体直链可用性。
- 编辑器原本被 App 外层 `ScrollView` 包住，底部面板没有自己的滚动容器，导致底部滑动手势会传给整页。
- 首次修复问题 #5 后，上方预览区固定且内容过高，在小屏幕模拟器上会挤占下方工具区，因此改为上下两个独立滚动区。
- 底部文字面板、图层和属性面板此前一直渲染在所有工具 tab 下，导致用户认为素材分类没有过滤。
- Bottom Sheet 初版已实现动画和拖拽关闭，但高度、遮罩透明度和工具栏图标仍需要模拟器视觉微调。
