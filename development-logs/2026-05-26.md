# 2026-05-26 开发日志

## 今日完成

- 继续排查元素无法自由拖动、四角按钮总是触发删除的问题。
- 定位根因：触摸事件落在文字、图片、贴纸或四角图标子元素上时，`locationX` / `locationY` 会变成子元素局部坐标，导致四角按钮判断总是偏向左上角。
- 修复触摸命中：将画布元素内部显示层和四角视觉按钮设置为不抢占触摸事件，让外层元素容器统一接收触摸坐标。
- 将右上角操作从“点击旋转”升级为“拖拽自由变换”：按住右上角向外拖动可放大，向内拖动可缩小，同时根据手指绕元素中心的角度调整旋转。
- 修复缩放状态保存：`handleResizeElement` 现在会同步保存 `frame.rotation`，保证自由变换产生的角度不会被状态层丢弃。
- 修复自由拖动仍不稳定的问题：画布元素交互开始时会临时禁用上方预览滚动区，避免父级 `ScrollView` 抢走拖拽手势。
- 优化右上角自由变换流畅度：缩放/旋转移动过程改为按动画帧节流更新，松手时刷新最后一帧，减少连续 `setState` 带来的卡顿。
- 修复贴纸本体不随外框变大的问题：emoji 贴纸字号现在会根据元素宽高动态计算，缩放时贴纸视觉内容和外框同步变化。
- 继续修复画布手势：移除画布外层预览 `ScrollView`，避免滚动容器在手势开始阶段抢占元素拖拽和旋转事件。
- 继续修复画布手势：为图片、文字、贴纸元素外层设置 `collapsable={false}` 并添加几乎透明的命中背景，确保外层元素容器可以稳定接收触摸事件。
- 修复 Bottom Sheet 遮罩阻断画布手势的问题：添加贴纸、选择图片、应用文字后会自动收起素材面板，避免半透明遮罩继续覆盖画布导致元素无法拖拽或旋转。
- 继续修复画布拖动/旋转移动事件丢失：元素 `PanResponder` 改为捕获阶段接管开始和移动事件，并阻止父级或原生控件中途抢走 responder。
- 用户安装 `react-native-gesture-handler` 后，已将手账元素交互从 `PanResponder` 迁移到原生手势库：元素内容区使用 `Gesture.Pan()` 负责自由拖动，右上角按钮使用独立 `Gesture.Pan()` 负责旋转和缩放。
- 已在 App 根部加入 `GestureHandlerRootView`，保证手势库可以正常接管 iOS/Android 原生触摸事件。
- 四角按钮改为独立手势区域：删除、透明度、复制使用 `Gesture.Tap()`，右上角变形使用拖动手势，避免一个父级触摸层继续误判所有按钮。
- 为手账编辑器顶部新增“返回首页”按钮，解决编辑器底部工具栏遮挡 App 主导航时无法切回主页的问题。
- 按参考图优化手账编辑器顶部布局：进入编辑器后隐藏 App 外层“情侣互动手账 / 手账 / 樱粉”标题区和主底部导航；编辑器内部不再显示“手账编辑器”标题，只保留返回首页入口。
- 将对齐辅助、回退、前进放到画布上方同一排并靠左显示，去掉画布外层卡片边框，让手账内容页占据更主要位置。
- 取消编辑器底部“保存到本机 + 状态提示”整行，将保存入口改为页面右上角单独“保存”按钮。
- 放大移动端手账纸张基础尺寸：按手机屏幕宽度生成 3:4 画布，让手账内容页更充分占据中间创作区域。
- 新增文字二次编辑入口：点击画布上的文字块后，会自动切换到“文字”工具并弹出文字编辑面板，可继续修改内容、字号、字体和颜色。
- 调整文字应用流程：点击“应用文字”后不再自动收起面板，方便继续微调文字样式。
- 优化文字选中外框：新增文字尺寸估算逻辑，点击文字、修改内容、调整字号/字形/颜色、切换字体后，文字元素外框会按文字内容自动收紧到合适大小。
- 根据参考图优化元素选中态四角控制点：选框改为灰色虚线，四角按钮改为粉色圆形底、白色线性符号，弱化原 emoji 风格。
- 拆分文字移动和文字编辑手势：单击/拖拽文字只负责选中与移动，双击文字才打开文字编辑面板，避免拖动文字时误弹编辑面板。
- 尝试进行模拟器回归测试：当前本机 `simctl` 能访问 Xcode，但没有可用 iOS Runtime 和模拟器 Device；Android `adb/emulator` 也不可用，因此无法完成真实模拟器手势回归。
- 尝试启动 Expo Web 做界面冒烟测试，但当前 Node 24 环境触发 Expo 端口探测异常，Web 预览未能启动。
- 已完成替代检查：TypeScript 类型检查通过；代码级回归确认编辑器页导航隐藏、右上角保存、文字双击编辑、元素拖动/缩放/复制/删除入口连接正常。
- TypeScript 类型检查通过。

## 遇到问题

- 模拟器表现说明此前的四角热区修复还不够，真正影响坐标判断的是子元素触摸目标不一致。
- 右上角原逻辑在按下时立刻触发单次旋转，不适合承载连续缩放和旋转手势，因此新增独立的 `transform` 手势模式。
- 画布在预览滚动区内部，父级滚动容器可能截获垂直拖动，导致元素自由移动不稳定。
- emoji 贴纸此前使用固定字号，导致只改变元素外框时，本体看起来没有缩放。
- 将内部内容层设为不接收触摸后，外层元素容器需要强制保留为 native view，否则触摸命中可能仍不稳定。
- Bottom Sheet 打开时遮罩会覆盖画布；如果用户添加元素后面板仍打开，画布元素无法接收拖拽和旋转事件。
- 删除、复制和透明度可以触发，说明按下事件正常；拖动和旋转无效说明移动事件可能被父级或原生 responder 抢走。
- 继续给 `PanResponder` 打补丁收益较低，已改为 `react-native-gesture-handler` 的原生手势模型；后续问题应优先围绕独立手势层、手势冲突和命中区域排查。
- 编辑器内部底部工具栏与 App 主底部导航同时存在，容易产生层级遮挡；当前先用顶部返回入口保证可退出编辑器，后续可考虑编辑器页隐藏 App 主底部导航。
- 编辑器页已隐藏 App 主底部导航，后续如果需要从编辑器直达“文件夹/共享”，需要在编辑器顶部或更多菜单中补入口。
- 保存状态提示暂时不再常驻显示，后续可考虑用轻提示或顶部短暂状态代替，避免挤占画布空间。
- 文字块点击会打开底部编辑面板，如果后续觉得拖动文字时误弹面板，需要进一步区分“轻点编辑”和“拖动移动”。
- 文字框尺寸目前使用内容和字号做估算，后续如需更精准，可接入真实文本测量或 `onTextLayout` 反馈。
- 四角控制点本次只调整视觉样式，删除、旋转缩放、透明度、复制的功能映射保持不变。
- 文字双击编辑基于手势库的双击识别，需在模拟器确认双击节奏是否自然；如偏难触发，可继续放宽双击距离或时间窗口。
- 当前机器缺少可用 iOS Runtime/模拟器 Device，无法直接做真机级手势验证；需要先在 Xcode 中安装 iOS Simulator Runtime 并创建/启动一个 iPhone 模拟器。
- Expo Web 在 Node 24 下启动失败，后续如需 Web 冒烟测试，建议切到 Expo 更兼容的 Node LTS 版本。
